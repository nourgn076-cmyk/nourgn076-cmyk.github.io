# How to ship this (5 steps, ~15 min)

## 1 — Create the repo
On GitHub, create a new repo named exactly:
`your-username.github.io`
(replace `your-username` with your actual GitHub handle — this exact naming is what makes GitHub Pages auto-serve it)

## 2 — Add these files
Push `_config.yml`, `index.md`, and `assets/css/style.scss` to the root of that repo.

## 3 — Personalize
Open `_config.yml` and replace:
- `author`, `email`, `linkedin`, `github_username`

Open `index.md` and replace the 3 placeholder project blocks with your real pinned repos (name, one-line problem, dataset, headline result, link).

## 4 — Turn on Pages
Repo → **Settings → Pages** → Source: **Deploy from a branch** → Branch: **main** / root → Save.
Your site goes live at `https://your-username.github.io` in ~1 minute.

## 5 — Make it yours (optional polish)
Ask Copilot/Claude: *"Suggest 3 small CSS tweaks to `style.scss` for a more distinctive minimalist look — keep it subtle, no more than one accent color."* Review and test every suggestion before committing (slide 22: verify, don't paste blindly).

---

**Checklist before you consider it done** (slide 24):
- [ ] Profile README on your GitHub profile (pro + perso, separate from this site)
- [ ] 4–6 best repos pinned and linked here
- [ ] Every linked repo has an honest README (problem, dataset, method, result, how to run, limitations)
- [ ] AI use disclosed
- [ ] Your point of view visible — not just "what" but "why"
